Go 基础面试题 —— slice 的切片、append、range

April 1, 2019 Mr9esx

1 minute read

执行以下代码时将打印什么？
package main
    
import (
    "fmt"
)

func main() {
    s := []int{1, 2, 3}
    ss := s[1:]
    ss = append(ss, 4)

    for _, v := range ss {
        v += 10
    }

    for i := range ss {
        ss[i] += 10
    }

    fmt.Println(s)
}

基础知识点：slice 是引用类型； append 在 len 大于 cap 的时候会创建新的 slice 执行值拷贝操作；range 的 value 是值拷贝。

输出：

[1, 2, 3]

原因： 首先对 s 进行了切片操作，因为切片操作两个 slice 其实都是共享同一个 array 数组，所以此时对 ss 的元素进行修改， s 也会跟着变化。

然后执行了 append 操作，此时 ss 内 array 的内存指向已经不是 s 的 array 了，所以后面的两个 for 循环都无法对 s 的数据进行更改。

如果把 ss = append(ss, 4) 去掉，那么只有第二个 for 循环能修改值，输出为：[1 12 13] ，因为 range 的 value 是值拷贝，直接对 value 操作不会影响 ss 和 s 的值，第二个 for 循环使用索引来操作，所以可以修改成功。
